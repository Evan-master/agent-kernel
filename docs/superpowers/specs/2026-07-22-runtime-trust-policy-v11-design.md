# Runtime Trust Policy V11 Design

Status: Approved for implementation on 2026-07-22

## Objective

Move signed-package trust from an x86 boot constant into deterministic kernel
state. V11 gives a native Agent a capability-authorized, generation-guarded
rotation transaction and proves that the resulting policy controls a later
ring-3 image admission.

The Resource Manager remains the initial signed Package v3. During its native
execution it rotates trust to a second signer. A deferred reuse Worker is
signed by that replacement key and is admitted only after the rotation.

## Layer Ownership

| Layer | V11 responsibility |
| :--- | :--- |
| `agent-kernel-core` | Fixed-capacity signer records, policy generation, atomic transition, Events |
| `agent-kernel` | Syscall-style trust and rotation facade |
| `agent-kernel-x86_64` library | Package signature arithmetic, call decode, typed message contract |
| `agent-kernel-x86_64` binary | Boot trust import, native call execution, QEMU proof |
| external tooling | Private-key use and deterministic Package v3 generation |

Private signing keys remain outside every kernel layer and outside the
repository.

Signing tools require private-key files with no group or other permissions and
reject output paths that resolve to the key itself. Reproducible package
evidence compares regenerated bytes with the package embedded in the kernel.

## Kernel Trust Record

`KernelCore` owns a distinct fixed array of signer records. Its capacity equals
the `AGENT_IMAGES` capacity for the current profile: every retained signer can
therefore authorize at least one retained image. Signer-record compaction can
decouple historical rotation depth in a later milestone.

Each `AgentImageSignerRecord` contains:

| Field | Contract |
| :--- | :--- |
| signer ID | 32-byte domain-separated SHA-256 identity |
| policy resource | Resource governed by the authorizing Capability |
| public key | 32-byte Ed25519 verification key |
| image-kind scope | Non-empty mask over Worker, Verifier, Fault Handler, Supervisor |
| ABI interval | Inclusive nonzero minimum and maximum |
| status | `Active` or `Revoked` |
| generation | Policy generation that last changed this record |

Signer ID derivation remains:

~~~text
SHA-256("AGENT_KERNEL_ED25519_SIGNER_V1\0" || ed25519_public_key)
~~~

The kernel validates canonical scope and ABI fields and computes signer IDs
itself. The loader remains responsible for Ed25519 public-key parsing and
signature arithmetic.

## Initial Trust

`trust_agent_image_signer` requires:

- an active actor;
- one non-revoked Capability on the policy resource;
- `Operation::Verify`;
- a previously unseen derived signer ID;
- valid scope and ABI bounds;
- one free signer slot and one free Event slot.

Success increments the global policy generation once, stores one `Active`
record, and appends `AgentImageSignerTrusted`.

The x86 boot flow imports the Resource Manager public anchor through this same
transition. Its resulting policy generation is `1`; there is no hidden trust
store mutation.

## Atomic Rotation

`rotate_agent_image_signer` receives:

- actor, authority, and policy resource;
- expected policy generation;
- previous signer ID;
- replacement public key, scope, and ABI interval.

The same Capability must allow both `Operation::Verify` and
`Operation::Rollback`. The transition rejects:

- stale policy generation;
- missing, cross-resource, or non-active previous signer;
- replacement identity equal to or already present in the store;
- invalid scope or ABI bounds;
- signer-store or two-Event capacity exhaustion.

All validation and capacity checks occur before mutation. Success performs one
transaction:

1. increment policy generation exactly once;
2. append the replacement as `Active` at the new generation;
3. mark the previous signer `Revoked` at the same generation;
4. append `AgentImageSignerTrusted` for the replacement;
5. append `AgentImageSignerRevoked` for the previous signer.

Replay order activates the replacement before revoking the previous signer.
Every error leaves records, generation, and Event sequence unchanged.

## Event Evidence

`Event` gains one optional `AgentImageSignerEvent` value containing:

- signer ID and optional rotation peer ID;
- public key;
- image-kind scope;
- ABI minimum and maximum;
- resulting status;
- policy generation.

Trust Events also bind actor, policy resource, Capability, and operation. The
canonical Event archive format advances to version `2` and hashes every new
field explicitly.

## Dynamic Loader Policy

`AgentImageTrustPolicy` becomes a borrowed view over kernel-owned signer
records. Signed Package v3 verification retains the V10 order:

1. canonical package parse;
2. verified record and complete-package digest agreement;
3. exactly one matching signer ID;
4. active status, policy resource-independent image kind, and ABI scope;
5. signer-ID derivation agreement;
6. `Ed25519::verify_strict` over the canonical signed prefix.

A revoked signer blocks future verification and admission. Already running
Agents retain their loaded frames; V11 does not asynchronously terminate an
execution context after policy rotation.

The Runtime Admission Broker converts any package, digest, signature, signer,
scope, or ABI verification failure into `RuntimeAdmissionFailure::ImageVerification`
before physical allocation. It appends one rejection Event, leaves the run
queue and frame pools unchanged, and prevents an invalid request from retaining
the head of the admission queue.

## Native Agent Call 53

Wire operation `53` is `RotateAgentImageSignerFromMemory`.

Register request:

| Register | Value |
| :--- | :--- |
| `rax` | `AGNTCALL` magic |
| `rbx` | ABI version `1` |
| `rcx` | operation `53` |
| `rdx` | flags `0` |
| `rsi/rdi/r8/r9` | scheduled Agent, Task, Image, nonce |
| `r10` | nonzero typed-message generation |
| `r11..r15/rbp` | `0` |

The kernel authenticates scheduled identity before snapshotting the private
call-data page. Typed message kind `2` keeps the existing 160-byte envelope:

| Offset | Bytes | Field |
| ---: | ---: | :--- |
| `0` | 8 | `AGNTMSG1` |
| `8` | 8 | message version `1` |
| `16` | 8 | message generation |
| `24` | 8 | kind `2` |
| `32` | 8 | total bytes `160` |
| `40` | 8 | payload bytes `112` |
| `48` | 8 | authority Capability ID |
| `56` | 8 | expected policy generation |
| `64` | 32 | previous signer ID |
| `96` | 32 | replacement public key |
| `128` | 8 | replacement image-kind scope bits |
| `136` | 2 + 2 + 4 | minimum ABI, maximum ABI, reserved zero |
| `144` | 8 | flags `0` |
| `152` | 8 | reserved `0` |

Success reply preserves the standard identity frame and returns:

| Register | Value |
| :--- | :--- |
| `rcx` | status `0` |
| `rdx` | operation `53` |
| `r10` | resulting policy generation |
| `r11` | retained signer count |
| `r12` | appended Event count `2` |
| `r13` | previous status `2` / Revoked |
| `r14` | replacement status `1` / Active |
| `r15/rbp` | `0` |

## Native Rotation Proof

The Resource Manager receives `Verify` authority on the boot policy resource.
Its ring-3 code writes the canonical kind-2 message and performs call `53`.

The deferred reuse Worker migrates from Capsule v1 to signed Package v3. It
retains its code transcript and gains one read-only byte so the segmented
format remains canonical. Its package is signed by the replacement key.

After Resource Manager completion, the kernel proves:

- policy generation `2`;
- one revoked initial signer and one active replacement signer;
- two adjacent rotation Events with complete replay fields;
- re-verification of the Resource Manager package fails as revoked;
- strict verification and admission of the reuse Worker succeeds;
- all existing address-space reclamation and Event archive checks still pass.

Successful QEMU completion emits:

- `AGENT_KERNEL_NATIVE_SIGNER_ROTATION_OK`;
- `AGENT_KERNEL_NATIVE_REVOKED_SIGNER_REJECTED_OK`;
- `AGENT_KERNEL_NATIVE_ROTATED_SIGNER_ADMISSION_OK`.

## Verification Gates

- Core tests cover initial trust, authorization, generation, duplicate IDs,
  invalid scopes, capacity, atomic failure, Event ordering, and archive digest.
- Facade tests prove the same transitions remain inaccessible through direct
  core mutation.
- Typed-message tests lock every byte, reserved field, and generation guard.
- Agent Call tests lock operation `53`, identity authentication, and reply
  encoding.
- Loader tests prove live policy acceptance and revoked-policy rejection.
- Package audits reproduce both signed images from external keys and verify
  assembly bytes, signer IDs, signatures, digests, and Release ELF uniqueness.
- Workspace tests, Supervisor, `no_std`, strict Clippy, debug QEMU, and Release
  QEMU pass.

## Deferred Work

- persistent signed trust state and rollback-resistant generation storage;
- signer-record retirement and compaction;
- multiple concurrent policy resources and quorum rotation;
- hardware-rooted boot trust;
- signing the remaining boot inventory;
- policy-driven termination or quarantine of already running Agents;
- SMP synchronization and TLB shootdown.
